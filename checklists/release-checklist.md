# XHS Release Checklist

Use this before publishing a Xiaohongshu post package generated from Obsidian notes.

## Source

- [ ] The post is based on a real note, case, transcript, course section, or repeated question.
- [ ] The source has one main topic, not several unrelated ideas.
- [ ] Sensitive names, private files, and internal paths have been removed.

## Title

- [ ] The title names a concrete reader, pain point, result, or method.
- [ ] The title is not only a broad topic such as `AI workflow` or `Obsidian tips`.
- [ ] The title does not promise views, followers, income, or platform results.

## Body

- [ ] The opening enters the reader's problem directly.
- [ ] The body contains concrete actions, not only general opinions.
- [ ] The final paragraph gives a natural next step.
- [ ] The draft does not expose tool internals, workspace paths, API keys, or private data.

## Carousel

- [ ] Each page has one job.
- [ ] The cover has a clear headline and visual focus.
- [ ] The middle pages show problem, method, and example.
- [ ] The final page clarifies fit, boundary, or next action.

## Image Prompts

- [ ] The image prompt describes layout, text hierarchy, visual elements, and aspect ratio.
- [ ] The prompt is specific enough for a designer or image model to follow.
- [ ] The prompt does not rely on vague words like `高级感` without concrete visual instructions.

## Archive

- [ ] The final title, body, tags, image prompts, images, and source note are saved together.
- [ ] The folder name includes the title, ratio, and date.
- [ ] The package can be reopened later for review without guessing how it was made.
