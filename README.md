# XHS Obsidian Workflow Starter

Free starter templates for turning Obsidian notes, Xiaohongshu saved notes, and X / Twitter source posts into Xiaohongshu post packages.

小红书收藏整理 / X 推文沉淀 / Obsidian 模板 / RedNote content workflow 免费 starter：先复制内容 Brief、收藏整理模板、X/Twitter 素材模板和发布包模板，再决定是否需要 79.8 元完整工作流包。

This repository is for creators who already keep drafts, course notes, interview notes, or reusable ideas in Obsidian, but still rebuild the Xiaohongshu / RedNote publishing workflow by hand every time.

## What This Starter Gives You

- A reusable Obsidian note template for one Xiaohongshu post.
- A collection inbox template for turning saved Xiaohongshu notes into source cards.
- A content brief template for turning one source note into title candidates, carousel plans, captions, and review checks.
- An X / Twitter source brief template for turning public high-signal posts into Obsidian source cards and Xiaohongshu content angles.
- A release checklist for title, body, carousel pages, image prompts, tags, and publishing order.
- A sample output structure for keeping every post package easy to review later.
- A simple boundary: this is a workflow scaffold, not a traffic guarantee or account operation service.

## Quick Start

1. If your Xiaohongshu saved notes are messy, copy `templates/xhs-collection-inbox.md` into your Obsidian vault first.
2. Move 5-20 saved notes into source cards and tag them as `usable`, `reference`, or `discard`.
3. If the source is from X / Twitter, copy `templates/x-twitter-to-xhs-brief.md` and record the public source, metrics, insight, reader translation, title candidates, and carousel plan.
4. Copy `templates/xhs-content-brief.md` when one source card is ready to become a post angle.
5. Move the final angle into `templates/xhs-post-brief.md` for title, body, carousel, and image prompts.
6. Use `checklists/release-checklist.md` before generating images or publishing.
7. Archive the final post package with the structure in `examples/sample-output-structure.md`.

## Recommended Workflow

```text
Saved Xiaohongshu note
  -> collection inbox
  -> source card
  -> content brief
  -> title candidates
  -> body draft
  -> carousel page plan
  -> image prompts
  -> publishing draft
  -> archive package
```

Each post should stay traceable. When a post performs well or badly, you should be able to reopen the folder and see the original source, the title decision, the image prompts, and the final draft.

- You already use Obsidian for notes, drafts, course material, or content ideas.
